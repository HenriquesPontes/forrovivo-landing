## 6. DATA SECURITY

### 6.1 How We Protect Your Data

**Encryption:**
- ‚úÖ **In Transit:** TLS 1.3 (industry standard)
- ‚úÖ **At Rest:** AES-256 encryption for database
- ‚úÖ **Passwords:** Bcrypt hashing (never stored in plain text)
- ‚úÖ **Tokens:** Stored in iOS Keychain (most secure storage)

**Infrastructure Security:**
- ‚úÖ SSL certificate pinning (prevents man-in-the-middle attacks)
- ‚úÖ Rate limiting (prevents API abuse)
- ‚úÖ Firewall and DDoS protection
- ‚úÖ Regular security audits
- ‚úÖ Automated vulnerability scanning

**Access Controls:**
- ‚úÖ Row-Level Security (RLS) on database (users can only access their own data)
- ‚úÖ Two-factor authentication for admin accounts
- ‚úÖ Least-privilege access (employees only see what they need)
- ‚úÖ Activity logging and monitoring

**Data Isolation:**
- ‚úÖ Each user's data is isolated in the database
- ‚úÖ Cannot access other users' data (enforced at database level)

**Incident Response:**
- ‚úÖ 24/7 security monitoring

- ‚úÖ Incident response plan
- ‚úÖ Regular backups (encrypted)

- Limit how we use your data

- Export your data in machine-readable format
